## 卡点预判

1. **pyOCD --erase chip 在某些 GD32 上可能失败**（GD32 flash sector size 与 STM32F103 略有差异——F103 标 1KB sector，GD32F303 是 2KB sector）。若失败用 `--erase auto`。
2. **复位后 LED 不闪**：可能原因 (a) main.c 启动卡在 SystemInit / 时钟初始化（HXTAL 起振失败？）；(b) GPIO 初始化错（虽然 KIT 抄的应该对）；(c) Flash 烧错位置（地址不对）。诊断：先 halt 看 PC、看是否进 main、栈指针是否在 RAM 范围内。
3. **Windows 上 pyOCD `--target stm32f103rc` 不在 builtin 列表的话**需 `pyocd pack install stm32f103rc`——不过我已确认 stm32f103rc 是 builtin（`pyocd list --targets` 看到了）。
4. **板子双供电冲突**：板子已通过 USB Type-C 供电（在 DFU 模式下），ST-Link 又通过 SWD header 提供 3V3——可能并联出问题。建议 SWD 不接 3V3，只接 SWDIO/SWDCLK/GND/NRST。pre 里写明这点。
