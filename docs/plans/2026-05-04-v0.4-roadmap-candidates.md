# v0.4 路线图候选 — 等用户拍板

## 当前状态（v0.3 完成）

**已 ship**：
- m1 PC host simulator（同源 host + MCU 跑通）
- m2 代码层分离（desktop_logic / themes/*.c / main wire）
- m3 软装数据化（.tstyle 自装到 SD + 加载）
- m4 golden image 自动验收（v0.2 北极星另一半）
- 多 size 字体 SD 加载（14/18/24/28/32）
- 真 RTC 接入
- 4 主题（TERMINAL/PIXEL/ZEN/NEON）+ baseline lock

**当前能力**：
- 改 theme 代码 / .tstyle → host 5s 闭环 → auto-verify PASS/FAIL → MCU 同源烧
- 13 → 4 用户介入次数（v0.1 → v0.3，降 70%）
- MCU Flash 98.52% 紧但能装；下个硬件升级解锁

## v0.4 候选方向

### A. 硬件升级 + Flash 释放
等用户给新 MCU（更大 Flash/RAM），重新 bring-up。释放 Flash 让 LV_USE_* widget 全开、所有 5 size 字体嵌入、5+ 主题、动画全开。
- 时间：1-2 天
- 依赖：用户硬件到货

### B. 图片资源 SD 加载 + 真·multi-resource theme
LVGLImage.py 可用（pypng+lz4 已装）。生成 PNG → .bin RGB565A8 → SD 卡。某个 theme 加 SD 加载的 logo/icon image。完整 dogfood "用 SD flash 资源"。
- 时间：2-3 小时
- 价值：图标/背景/动画帧 SD 加载是嵌入式 GUI 高级能力

### C. 多页面架构
当前是单一页面 5s 切 theme。升级到：launcher 页（4 个 theme 缩略图选择）+ details 页 + settings 页。状态机驱动。无按键所以"页面切换"还是 timer 触发，但架构成熟。
- 时间：1 天
- 价值：演示真实 watch UI 的页面架构

### D. 真传感器接入
fake batt sweep / fake hr → 真 INA219（电池）/ 真 MAX30102（心率）/ 真 BMI160（步数）。需要硬件接入 + I2C bring-up。
- 时间：每个传感器 1-2 小时
- 依赖：传感器模块到货

### E. 动画 + 过渡
当前主题切换硬切。加 fade-in / slide-in / arc sweep 动画。LVGL anim API 支持，但需要小心 OOM。
- 时间：半天
- 价值：精雕用户体验，也是软装迭代周期的延续

### F. Capability Report v0.4 + 完整 retro
不写代码，写产出。把 v0.1 → v0.3 全部经验整理成方法论文档：Playbook + 模板 + lessons → 最终 deliverable。
- 时间：半天
- 价值：项目元目标的真正交付（手表是 case，方法论才是 product）

## 我的建议优先级

**短期（用户硬件升级前）**：
1. **F**（capability report 收口）— 最低风险、产出元目标的核心交付物
2. **B**（图片 SD 加载）— 用户原话强调"用 SD flash 资源做开发"，dogfood 直接对齐
3. **E**（动画）— 软装迭代延续，精雕

**长期（硬件升级后）**：
4. **A**（重新 bring-up 新硬件）
5. **D**（传感器接入）
6. **C**（多页面架构）

**不建议**：直接堆 feature。当前 sk demo 已经能展示完整能力，再加一两个 theme/widget 边际效益递减。
