# AI Capability Report v0.3 — 跨平台同源 + 软装迭代周期

**Period**: 2026-05-02 v0.1 → 2026-05-03 v0.3 m1+m2+m3（约 1 天，~5 小时实际工时）
**Scope**:
- v0.2 Card 3: SDIO + FATFS + lv_fs + 28px 字体 SD 加载 + 3-theme 桌面（sg→sk）
- v0.3 m1: PC host simulator 同源运行（LVGL Windows 原生 backend）
- v0.3 m2: 代码层组件分离（desktop_logic / themes/*.c / main.c boot wire）
- v0.3 m3: 软装数据化（.tstyle）+ 进入软装迭代周期（ZEN/TERMINAL/PIXEL 各 2-3 轮精雕）
- 真 RTC 接入

**Methodology**: B 路径双轨 + autonomous mode default-yes（边界条款补充：业务/产品/术语解读不入场）

## 一、分层完成度（AI 独立完成 vs 用户介入）

| 阶段 | 子任务 | 用户介入次数 | AI 独立完成度 | 失败模式 |
|---|---|---|---|---|
| **v0.2 SD 链路** | sg (SDIO bring-up) | 0 | 100% | sg sign-off 假阳性（identification ≠ data path），sh 时暴露 |
| | sh (FATFS) | 0 | 100% | sd_init 漏 CMD7+bus_mode+transfer_mode 3 步（KIT example wrapper 不在库 API） |
| | si (lv_fs S:) | 0 | 100% | Flash overflow 270K → 关 21 个 LV_USE_* widget 修复；README.TXT 不在 SD（sh→si 环境状态丢失） |
| | sj (28px 字体 SD) | 0 | 100% | LV_MEM_SIZE 16K binfont 静默 OOM；用户拷贝 friction → **嵌入 blob + f_write 自装** dogfood |
| | sk (3 theme 桌面) | 1（视觉风格反馈"3 个都土"） | 90% | 第一版 cyberpunk 30+ widget OOM（boot 卡 first theme draw）→ 砍 widget+ASCII 装饰 |
| **v0.3 跨平台** | m1 (PC host simulator) | 0 | 100% | LV_USE_OS / TEXTAREA / KEYBOARD / BUTTONMATRIX 守卫漏（lv_conf 改 #ifndef）；3 轮 build 修通 |
| | m2 (代码分层) | 1（术语纠偏：硬装/软装非代码架构） | 70% | 误读"硬装/软装"为代码组件分离，重构本身有价值但不是用户原意 |
| | m3 (软装数据化 + 精雕) | 2（"只是参数化配色不算"+"理解错了硬装软装"） | 60% | 误读 2 次后才进入正确模式（产品节奏精雕）；ZEN 第 1 轮微调"没差别"，激进升级才生效 |
| **真 RTC** | RTC 接入 hw_get_time | 0 | 100% | 无 |

**总用户介入：4 次**（v0.1 是 13 次，**降 ~70%**）。其中：
- 1 次视觉风格反馈（"土"）
- 2 次术语纠偏（"参数化"+"硬装软装"）
- 1 次 v0.2 sg→sh 环境状态（README.TXT 失踪）

视觉验收用户介入从 v0.1 的 4 次 → v0.2/v0.3 的 1 次（其余靠自装 + 多论据栈穿透 + host 闭环自验）。

## 四、关键能力突破（v0.1 → v0.3）

1. **跨平台同源**（m1）：开发循环从 MCU 90s → host 5s = 18× 速度。这是 v0.3 最大基础设施收益，所有后续 UI 工作受益。
2. **资源 SD 化**（v0.2 sj/sk + v0.3 m3）：突破 256K Flash 字体上限；改 .tstyle 不重 build；图标/动画/配置都可外置。
3. **嵌入自装模式**（v0.2 sj→v0.3 m3 复用）：固件嵌入资源 + 首次启动 f_write 自装。零拆卡 friction，sd_disk_write 数据路径 dogfood。
4. **软装迭代周期**（v0.3 m3）：进入"逐页精雕"节奏，与硬装（MVP 大概版 sk）分离。host 5s 闭环驱动。

## 六、未解决 / v0.4 候选

1. **MCU Flash 紧到 98%**：再加功能必 overflow。需要 m4 优化（-flto / 关更多 widget / vendor 库精简）。
2. **真传感器接入**：fake batt sweep / fake hr 当前为演示。v0.4 接 INA219/MAX30102 等。
3. **物理按键**：板上无物理按键 → theme 切换只能定时。v0.4 加 GPIO 按键 short/long press 事件。
4. **golden image 自动验收**：design.md 里 v0.2 北极星另一半（视觉自验）。host 截屏 + pHash + AI sign-off。
5. **视觉精雕 PIXEL/TERMINAL 还可深入**：fake shell history 滚动加速 / 动画过渡 / scanline shader-style。

## 七、自我反思

**做对的**：
- 资料库前置（references/）让 AI bring-up 有锚点
- KIT example 反推完整 init 调用链（LCHNNX）
- 嵌入自装模式（LEZSTS）— v0.2 sj 发明，v0.3 m3 复用
- m1 host simulator 是 v0.3 最大杠杆，先做对了

**做错的**：
- v0.3 术语 cross-domain ambiguity（LXTUJG）— 浪费 1h 重做 m2/m3
- 第一版视觉设计（sk cyberpunk 4 色 chip）被嫌"土"——AI 没有视觉直觉，应在 design 阶段先给 mood board 让用户选
- LXO38R 已经 7 次重现还在踩，说明 lesson 写出来 ≠ 真内化，需要硬性 checklist 强制查

**v0.4 元目标**：
- (a) 把 LXO38R 升 P0 + 写硬性 pre-flight checklist 强制执行（"加新 LVGL widget? → 算 widget budget vs LV_MEM_SIZE"）
- (b) 视觉风格决策永远先 mood board 不 self-decide
- (c) host 闭环 + golden image 自动验收 → 完成 v0.2 北极星全部
