# LXO384 — LVGL v9 在 256K Flash 上的最小可用配置
**Rule:** GD32F303RC (256K Flash) 上跑 LVGL v9.5.0：关 LV_USE_THEME_DEFAULT (省 26K) + LV_DRAW_SW_COMPLEX 仅在用 arc/spinner 时开 (开了占 15K) + 每个 Montserrat 字体 ~25K (一次只能开 1 个) + LV_USE_SYSMON 调试期开开发完关 (省 6K) + LV_MEM_SIZE 16K (8K 会 OOM 死循环)。
**Why:** sb 期间 default 配置 Flash 261K 超 256K → 反复裁剪 41K 才能跑；sd 期间想加 Montserrat 24/28 都超 256K。GD32F303RC 是 LVGL v9 落地的 size 极限平台，每个开关都是 trade-off。

**How to apply:** 起手就用本配置 baseline，之后只能减不能加。开 LV_FONT_MONTSERRAT_X (X>14) 必须删别的（关 LV_USE_SYSMON 是首选）。LV_MEM_SIZE 不要低于 16K（实证 8K + sysmon+spinner+arc 并行触发 TLSF 死循环）。

**Evidence:**
- sb 调 size 261K→242K
- sd Montserrat 24 仍超 256K
